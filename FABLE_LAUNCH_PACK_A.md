# ✍️ Fable Launch Pack A — 顧事 novel-web 寫作引擎

> **點用**：喺 Cowork 開一個 **Fable** session（唔使 mount 都得，內容係新寫嘅），
> 先做 **STEP 1** 萃取 voice profile（一次過），之後 **STEP 2** 大量生成。
> 設定 **effort = high**。俾佢「要乜」，由佢決定「點寫」——唔好淨係打「寫一篇連載」。
> Fable 係 Anthropic 最強 writer，質素會遠超而家用嘅 DeepSeek，所以呢兩日重點榨呢樣。

---

## 0. 產品 context（Fable 要知嘅硬規格）

- **品牌**：「顧事」——中文網絡爽文網站（https://novel-web-sepia.vercel.app ）
- **儲存**：Supabase `novel_stories` 表，欄位包括 `title` / `genre`（類別）/ `story_type` / `body`（全文，公開讀）
- **兩種 story_type**：
  - `serial`（連載爽文）：**1200–1800 字**，cliffhanger 收尾、**唔完整**、留鈎；讀者之後會揀分支生成個人化結局
  - `short`（完整短篇）：**1500–2200 字**，有頭有尾，**情感要夠深**（見下方硬性要求）
- **25 類別加權池**（男頻 weight 1 / 女頻 weight 2）——重生、系統流、都市異能、宮鬥、甜寵、玄幻、末世、種田、無限流、家庭倫理…（沿用現有池，Fable 可以自己補齊常見類別）
- **短篇情感四硬規**（Stephanie 曾反饋「太簡單」而加，務必守）：
  1. 揀定**單一主導情感基調**
  2. 用**具體場景／對白／物件**推進情感，唔准「他很感動」式概括
  3. 結尾要有**明確情感爆發點／淚點／會心一笑收筆句**
  4. 情感要有**轉折**，唔可以由頭到尾同一種情緒

## 1. Voice Profile 模板（你填 → 交畀 Fable 萃取）

> 填得越具體，Fable 定出嚟嘅 house style 越準。3–5 個例子最好。

**A. 我鍾意嘅範文／作品（貼片段或講書名 + 點解鍾意）**
1. ＿＿＿＿＿＿＿＿＿＿（例：《詭秘之主》開場——鍾意佢嘅懸念鋪陳同代入感）
2. ＿＿＿＿＿＿＿＿＿＿
3. ＿＿＿＿＿＿＿＿＿＿

**B. 想要嘅整體調性**（圈或補充）
- 爽度：□ 極爽快節奏  □ 中速有鋪陳  □ 慢熱後爆發
- 情緒底色：□ 熱血  □ 治癒  □ 虐心  □ 搞笑  □ 懸疑
- 文字風格：□ 白話直接  □ 畫面感強  □ 金句多  □ 冷幽默

**C. 一定要避（禁忌）**
- ＿＿＿＿＿＿（例：爛尾、突然開掛冇鋪墊、說教、性描寫）

**D. 讀者係邊個 / 睇嘅場景**
- ＿＿＿＿＿＿（例：返工通勤 5–10 分鐘一篇、想要爽 + 有記憶點）

> ⚠️ 如果你懶得填 B–D，直接淨係俾 A（範文），叫 Fable 自己反推 voice profile 都得——佢做呢樣好強。

## 1.5 STEP 0 — 零輸入起步（冇範文？由呢度開始）

> 唔使填 Section 1。直接貼呢個 prompt，Fable 出 3 個唔同 voice 各一小段，你揀中先鎖做 house style，之後再跳去 STEP 2 生成。

```
你係頂尖中文網絡文學編輯。我做緊一個爽文網站「顧事」（讀者返工通勤幾分鐘睇一篇，
要爽 + 有記憶點）。我暫時未有明確 voice 偏好，想用「試讀」方式揀。

請用 high effort 設計 3 個明顯唔同嘅寫作 voice（例如：A 極快節奏熱血爽 / B 畫面感強
帶懸念 / C 冷幽默都市感——你自己定，要夠分別），每個 voice：
- 一句風格定位
- 用同一個開場情境（主角踏入一個異常場面）各寫一段 150 字示範，令我感受到分別

出完 3 段之後停低，等我揀。我揀完你就將嗰個 voice 擴寫成完整 Voice Profile
（照 STEP 1 嗰 7 項），再等我落 STEP 2 生成。全程繁體中文。
```

## 2. STEP 1 Prompt — 萃取 Voice Profile（有範文／偏好先用；零輸入已由 STEP 0 覆蓋）

```
以下係我鍾意嘅網絡小說範文同偏好（見附）。你係頂尖中文網絡文學編輯。
請用 high effort，深度分析呢啲範文嘅共通「voice」，然後幫「顧事」呢個爽文網站
定出一份可重用嘅 Voice Profile（House Style），包含：

1. 敘事視角同人稱傾向
2. 句子節奏／段落長度／爽點密度（幾多字一個爽點）
3. 對白 vs 敘述比例、金句運用
4. 開場鈎子公式（頭 3 句點抓人）
5. 連載 cliffhanger 收尾公式 vs 短篇情感收筆公式
6. 用詞禁忌清單
7. 三個「一睇就知係顧事出品」嘅簽名特徵

輸出成一份精煉、可以直接塞落生成 prompt 嘅 Voice Profile（繁體中文）。
生成之前，如果你覺得我俾嘅範文唔夠定 voice，最多問我兩條關鍵問題。

【附：我嘅範文同偏好】
<< 貼上 Section 1 填好嘅內容 >>
```

## 3. STEP 2 Prompt — 大量生成旗艦內容（貼落 Fable）

```
用上一步定好嘅 Voice Profile。你係「顧事」嘅駐站作者，用 high effort 一次過寫出
一批可以直接上架嘅內容。硬規格：

連載 serial（寫 3 條，各 1200–1800 字）：
- 每條係一個獨立世界觀嘅第 1 章，cliffhanger 收尾、留強鈎
- 附：類別(genre)、書名(title)、一句 logline、後續 3 章走向大綱、3 個可俾讀者揀嘅結局分支種子

短篇 short（寫 3 篇，各 1500–2200 字）：
- 完整有頭有尾，嚴守情感四硬規（單一主導情感 / 具體場景推進 / 明確淚點或會心收筆 / 情緒有轉折）
- 每篇附：類別、書名、一句主導情感描述

輸出格式：每篇用清楚分隔，開頭一行 metadata：
[story_type=serial|short | genre=類別 | title=書名]
之後正文。方便我直接入 Supabase novel_stories。

要求：
- 6 篇之間類別／情緒要分散，唔好重複同一味
- 唔准爛尾、唔准無鋪墊開掛、唔准說教
- 全部繁體中文（用字自然，唔好翻譯腔）
先出 1 篇連載 + 1 篇短篇畀我睇，我 OK 你先一次過寫埋餘下 4 篇。
```
